v0.5.4

Released on 03/09/2021

A hotfix was released in the morning of the 4th of September, after it was discovered that a single missing exclamation mark on line 12 of Ownership.cpp, was causing the plugin to never assign squawk codes or set initial climbs to departing aircraft.

Features

  • Plugin will now (continuously) check your plugin and sector file versions and notify you if they're out of date

  • Colours for tag items and screen elements will change dynamically depending on your position (as with the recent TopSky Update)

  • New¬†Assign Local Squawk¬†option in the squawk assignment dialog to easily assign local circuit squawks.

  • Position of plugin screen elements (menu, metars, releases) can now be saved to ASRs

  • New Met √âireann Window features:

    • Updated METARs will be highlighted on the Met √âireann Window

  • New Oceanic Reclearance Flag will be shown as an¬†'O'¬†at the top of an aircraft's tag if there has been an amendment to their oceanic clearance

  • New¬†help,¬†commands,¬†settings¬†and¬†guide¬†commands in case you get lost while controlling

Bugfixes

  • Unresponsive APIs will no longer cause the plugin to crash

  • Plugin will no longer crash when switching positions or unpriming and repriming frequencies

  • Fixed bug where positions that don't control any airports top-down (FIN, DWUS, etc.) weren't shown any METARs

  • Aircraft passing through an oceanic entry point on a random routing will no longer be assumed to be on a NAT

  • Fixed bug where plugin would never send data requests because it tried sending them before websocket was connected

  • Stopped plugin from crashing when the websocket is down

  • Fixed bug where plugin didn't know who to send a release to when it was accepted before being requested

  • Fixed some issues with the RFL validator logic

  • Fixed bug where waypoints with the format XXXXX/N####F### were not recognised

  • Plugin will no longer leak paintbrushes slowly causing ES to freeze and/or become utterly f*cked during long sessions

  • Aircraft data will now only be deleted if the aircraft disconnects after landing or outside the airspace

  • Changed notification for outdated plugin/sectorfile to a popup which opens the appropriate update forum post

  • Renamed AutoPDC to SmartPDC to reduce confusion. The setting is also renamed to¬†smart_pdc

  • Refactored vIAC Squawk Assignement:

    • Squawks are now based on the aircraft registration, not the callsign.

    • vIAC must now have¬†VIRTUALIAC.COM¬†in the remarks to be assigned vIAC squawks

    • vIAC aircraft flying IFR overseas are assigned 66XX squawks

  • Simplified and corrected squawk assignment logic

  • When an ATIS is disconnected, the ATIS code will be shown in red on the METAR display and removed after 5 minutes

  • Refactored Colours:

  • Refactored data request logic so that only stations known to be connected to the websocket are sent data requests

  • Dublin Group controllers will now be shown METARs from all available Dublin Group airports

  • Updated Documentation (Changes marked with a blue line on the side)

API

  • Each plugin version now has its own API endpoint to allow for changes to be made during development without disruptions

  • Moved entire API to a new (temporary) home due to security concerns

  • New format for vIAC squawks for added flexibility

  • Added Bal SIDs (They were there already, just misspelled...)

  • Added new¬†*ENDEQ¬†and¬†*BOFUM¬†SIDs
