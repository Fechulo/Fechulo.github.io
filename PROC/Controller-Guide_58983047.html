Installation

  • Extract the zip file into your Euroscope Plugins folder (...\VATeir Euroscope Package\Plugins\VECP)

  • Get your API Token from the VATéir website and paste the token into the key.txt file in the plugin folder.

This token is unique for every controller. You should not share this with anybody or you might be held responsible for something you didn’t do.

  • Open EuroScope and navigate to OTHER SET -> Plug-ins -> Load and load in the VATeirControllerPlugin.dll file found in the plugin folder.

  • Ensure the plugin is allowed to draw on both the Ground Radar Display and the Standard ES Radar Screen.

General Functionality

The VATéir Controller Plugin introduces a number of features designed to assist all VATéir controllers.

Syncing of Aircraft Data

Aircraft data, such as clearance flags and ground states, is shared between plugins to keep everyone’s lists up-to-date.

Plugin Menu

The Plugin Menu can be opened by clicking on the icon found at the bottom right of EuroScope:

Through this menu, some general plugin settings can be changed and loaded/saved, visual elements can be turned on/off, TSAT options can be modified to fit the traffic situation, and plugin data can be reloaded in case of issues.

Chat Commands

The VATéir Controller Plugin introduces a number of commands that can be used through the EuroScope chat. For compatibility with aliases and other plugins, all VÉCP commands follow this format:

.VECP [command] (argument)

For more information on these, make sure to read the Commands Reference.

Settings

A settings.txt file can be added into the plugin directory, allowing for some customisation of the plugin’s appearance and functionality.

For more information on these, make sure to read the Settings Reference.

Timer

The VATéir Controller Plugin provides a timer which can be useful when applying departure separation or adhering to MDIs.

There are 3 buttons to easily start a timer for 30s, 60s and 120s. Click the C button to create a custom timer. If you wish to edit the previously entered custom timer, simply Right-click and enter a new value.

To reset the timer, click the R button at the top-left of the timer.

The timer can be hidden through the display settings or by clicking on the X button at the top right of the timer.

Met Éireann

Met Éireann Window

The VATéir Controller Plugin automatically downloads weather data for all Irish airports and displays the information relevant to you in a Met Éireann window.

When a METAR update is received, the relevant entry will be highlighted. It can be acknowledged by clicking on the entry or on the A button at the top left of the window.

Right-clicking on an entry in the display will open a page with more detailed aerodrome information, including the full METAR and wind components for each runway, latest TAF, and charts.

When an ATIS is found, its code will be shown on the left-most column of the Met Éireann window. When the ATIS is disconnected, the code will be shown in red for 5 minutes, after which it will be cleared.

The Met Éireann window can be hidden through the display settings or by clicking on the X button at the top right of the window.

Regional Pressures Window

Controllers covering a position that provides a Flight Information Service to aircraft will be shown a Regional Pressures Window which displays all the relevant regional pressures.

When a pressure change is detected, the relevant entry will be highlighted. It can be acknowledged by clicking on the entry or on the A button at the top left of the window.

The Regional Pressures Window can be hidden through the display settings or by clicking on the X button at the top right of the window.

Met Éireann Notifications

VÉCP will notify you of any changes in QNH and Transition Level at airports relevant to you.

These notifications can be disabled through the general settings.

Aerodrome Functionality

The VATéir Controller Plugin introduces a number of features designed to assist aerodrome controllers.

Squawk Assignment

VÉCP will automatically assign squawks to aircraft departing from Irish airfields.

  • IFR aircraft will be assigned squawks in the range 6600-6677. During periods of very heavy traffic, the plugin may widen this range so as to avoid duplicate squawks.

  • VFR aircraft will be assigned squawks in the range 0400-0477 inside the Dublin CTA, and 2600-2677 when outside.

  • Virtual Irish Air Corps aircraft will be assigned special squawks corresponding to their callsigns.

vIAC pilots must have OPR/VIRTUALIAC.COM in their remarks for this to work correctly.

  • Local circuit squawks will not be assigned automatically. Click on Local Set in the squawk assignment dialog to assign an appropriate local squawk.

Initial Climb Assignment

VÉCP will automatically assign initial climbs to aircraft departing from Irish airfields.

This is based on the SID filed by the pilot so if there is no SID, or if EuroScope cannot extract the SID from the flight plan, the plugin may not assign an initial climb.

Just because initial climbs are assigned by the plugin, does not mean controllers should stop checking these. If an A380 files a KISHA departure, the plugin will blindly assign 4000ft as the initial climb. Controllers should continue to carefully inspect flight plans and address any issues, before issuing a departure clearance.

Smart PDC

The default clearance aliases are overwritten by VÉCP to automatically include the correct airborne frequency. This feature can be turned off through the general settings.

Alias

Function

.pdc

PDC sent via private message

.clr

PDC sent via frequency chat

.sid

Text clearance for text pilots

.sidab

Text clearance for text pilots (including initial climb, for CAT A/B aircraft)

Departure List Items

Departure Frequency

VÉCP will calculate the correct airborne frequency for aircraft departing from Irish airfields and display it on the departure list.

This is based on the SID filed by the pilot so if there is no SID, or if EuroScope cannot extract the SID from the flight plan, the plugin may not be able to calculate the airborne frequency.

If an aircraft’s departure frequency changes AFTER the aircraft has been cleared, the new departure frequency will be shown in red. This indicates that the aircraft needs to be given the new frequency. Once the pilot is informed of the change, simply click on the red airborne frequency to turn it blue.

Departure frequencies can be manually overwritten by right clicking on the frequency and selecting a new controller. The tag item may display UNKNOWN for aircraft with unusual routes.

Flight Plan Validator

VÉCP will scan IFR Flight Plans for incorrect flight levels and SIDs. The possible indicators for invalid flight plans are shown in the table below:

Indicator

Meaning

OK

Flight plan is deemed valid

RFL

The requested flight level is invalid for the direction of flight

CAT

The requested SID is invalid for the aircraft category

UFO

The aircraft type is not recognised by the plugin. If you think it should be, let Nacho know

Just because VÉCP couldn’t find any problems with a flight plan, doesn’t mean there aren’t any. Controllers should continue to carefully inspect flight plans and address any issues, before issuing a departure clearance.

Electronic Releases

Controllers can coordinate departure releases electronically through VÉCP. The release tag item has three possible states which can be identified by the item’s colour on the departure list: set-up, requested and approved.

The release can be set up by any controllers (including the target releasing controller) by clicking on the departure list item and selecting a controller.

Although the target releasing controller can accept the release at this stage, they will not be notified of the release.

When the aircraft is close to departure, controllers on a TWR position (or higher) can commence the request by clicking on the red release item and clicking request.

At this point, the target releasing controller will be notified of the request and be prompted to either accept or reject the release.

If the release is accepted, there will be an audible indication, a message in the coordination chat, and the status will be shown in blue on the departure list.

CTOTs

VÉCP introduces CTOT functionality to aid Delivery Controllers during events with allocated slots.

The CTOT field will be automatically populated and shown in blue for aircraft that have an event slot. Non-event aircraft can be assigned a CTOT manually once their route is approved by the ECFMP.

If the aircraft is still on the ground by their CTOT, it will begin to flash red. The CTOT field will be automatically cleared when the aircraft is cleared for take-off and its ground state is updated.

TSATs

VÉCP introduces TSAT functionality to aid Delivery Controllers during events. Controllers may choose to assign TSATs to departing aircraft during periods of heavy traffic.

To assign one, simply click on the TSAT field and select either AUTO Assign to let the plugin calculate a suitable TSAT, or MAN Assign to type in a specific time.

Aircraft with assigned CTOTs will be given assigned a TSAT equivalent to CTOT-15.

The default TSAT interval used by the plugin is 5 minutes. This can be changed through the TSAT Settings in the Plugin Menu

If the aircraft is still on stand by their TSAT, it will begin to flash red. The TSAT field will be automatically cleared when the aircraft is cleared for push and/or start and its ground state is updated.

Area Functionality

The VATéir Controller Plugin introduces a number of features designed to assist area controllers.

Squawk Assignment

As with aircraft departing from Irish airfields, aircraft entering the Shannon FIR will also be assigned an appropriate squawk, unless already squawking a previously assigned discrete squawk.

See the Squawk Assignment section in the Aerodrome Functionality area for more information.

Sector Exit List Items

FETX

The FETX column in the sector exit list simply shows the estimated FIR exit time. For aircraft arriving into Irish airfields, this will display the estimated time of arrival, as calculated by EuroScope.

MNR

This is an aircraft’s cleared oceanic mach number, e.g., ‘M083’ = Mach 0.83.

VÉCP will attempt to populate this field automatically when an aircraft receives clearance through natTRAK or vNAAATS, but it can also be edited manually.

If a re-clearance is detected by the plugin, which contains a change the to the MNR, the item will be highlighted on the sector exit list. Clicking on the Oceanic Re-clearance Flag shown at the top of the aircraft’s tag will acknowledge the change.

OFL

This is an aircraft’s cleared oceanic flight level.

VÉCP will attempt to populate this field automatically when an aircraft receives clearance through natTRAK or vNAAATS, but it can also be edited manually.

If a re-clearance is detected by the plugin, which contains a change the to the OFL, the item will be highlighted on the sector exit list. Clicking on the Oceanic Re-clearance Flag shown at the top of the aircraft’s tag will acknowledge the change.

NAT

This is an aircraft’s filed North Atlantic Track (or Tango Airway).

VÉCP will populate this field automatically based on the published tracks and the aircraft’s flight plan.

During oceanic events with published tracks, the even tracks will be used instead.

If there is no track matching the aircraft’s flight plan, a ‘#’ symbol will be shown in its place symbolising a random routing.

OAN

This is an aircraft’s oceanic entry point.

VÉCP will populate this field automatically based on the published tracks and the aircraft’s flight plan.

During oceanic events with published tracks, the even tracks will be used instead.

Oceanic Entry Time

The CTO, NBT and NLT items display an aircraft’s cleared oceanic entry time.

due to limitations of the natTRAK and vNAAATS APIs, VÉCP cannot populate these fields automatically. A time can be entered manually instead into any of these fields.

The colour of the item indicates whether controller action is required to meet the time constraints

  • white indicates the aircraft is expected to enter oceanic airspace as per their clearance (within a 2 minute tolerance)

  • yellow indicates the aircraft is expected to enter oceanic airspace as per their clearance, but very close to the tolerated limits. Controller action may be required to ensure the time constraint is met.

  • red indicates the aircraft is expected to enter oceanic airspace outside of their clearance clearance. Controller action is required to ensure the time constraint is met.

CTO

This is the cleared oceanic entry time.

A + symbol means the clearance is for not before the cleared entry time

A - symbol means the clearance is for no later than the cleared entry time

NBT

The cleared entry time will be shown here if the cleared oceanic entry time contains a “not before” restriction.

NLT

The cleared entry time will be shown here if the cleared oceanic entry time contains a “no later than” restriction.

Examples

  • EXS541 is cleared to enter oceanic airspace at 1525z, since their FETX is within limitations, the time is shown in white.

  • VIR63 is cleared to reach SOMAX no later than 1527z. Since their FETX is also 1527z, the NLT is shown in yellow to warn the controller that action may be needed in order to meet this restriction.

  • AFR334 is cleared to reach BEDRA not before 1550z. Since their FETX is 1547z, the NBT is shown in red.

OCA Flag

The colour of the OCA flag is an indicator of the status of an oceanic aircraft’s oceanic clearance.

  • Red indicates the aircraft has not yet requested oceanic clearance.

  • Yellow indicates an aircraft has requested and is awaiting oceanic clearance (either through natTRAK or on Shanwick Radio).

  • Blue indicates the oceanic clearance has been issued,

If natTRAK and/or vNAAATS are in use, the OCA flag and other oceanic list items will be updated automatically. If neither natTRAK nor vNAAATS are in use, the OCA flag can be cycled manually by left clicking on it.

Right-clicking on the flag will print a message with some information regarding the status of that aircraft’s oceanic clearance. Some extra information may be shown here that the plugin was not able to process, including route changes, time restrictions or instructions directed at the pilot.

Tag Items

Intention Codes

Most aircraft tags will contain an intention code. These 3/4 letter codes act as an indication of the routing of a flight. The intention codes will differ depending on a number of different factors. The table below summarizes how they work:

Dublin ACC

Shannon ACC

Departures & Overflights

First 3 letters of the Dublin CTA exit point

First 3 letters of the Shannon FIR exit point

Arrivals

DW/NN/CK/ME: First 3 letters of the destination ICAO code

Others: Full destination ICAO code

DW/NN/CK/ME: First 3 letters of the STAR identifier

Others: Full destination ICAO code

Oceanic Re-clearance Flag

Oceanic Controllers will often issue amendments to an aircraft’s oceanic clearance. If a re-clearance is detected by the plugin through the natTRAK or vNAAATS APIs, an Oceanic Re-clearance Flag will be shown above the aircraft’s tag in the form of an ‘O’. The affected items will also be highlighted on the Sector Exit List. Clicking on the flag will acknowledge the change and hide the flag.

Request On Frequency

Pilots will often forget to contact their departure frequency when airborne. Unassumed departures, between 2000ft and FL90, who were assigned your frequency for departure, will display a Request On Frequency item above the aircraft’s tag in the form of an ‘ROF’. Clicking on this item will notify the acting Tower controller that the departure has not yet called you.
