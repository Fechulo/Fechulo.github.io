Introduction

A settings.txt file can be added into the same directory as the VATeirControllerPlugin.dll file, allowing for some customisation of the plugin’s appearance and functionality.

Each setting must be on its own line with a single space separating the setting name from the setting value.

If the setting value is made up of multiple elements (e.g. RGB colour value), they should be separated by a single comma.

The order and capitalization of settings is irrelevant; YoU cAN tYpE LIke ThIS if yOu lIke.

The settings file can be edited while EuroScope is running and the settings can be reloaded / saved through the plugin menu

Some changes may not take immediate effect

General Settings

Below is a table of all available settings, including a brief description of the setting and the default value.

See the appendix for a sample settings file.

Setting Name

Default

Description

debug

false

If enabled, debug information is printed to Euroscope.

This should be used for testing purposes only and is not recommended for general use.

sweatbox_mode

false

If enabled, VÉCP will behave on Sweatbox just as it does when connected to the network.

This should be used for testing purposes only and is not recommended for general use.

metar_notifications

true

If enabled, QNH and TL updates are sent when relevant METARs are updated.

metar_display

true

If enabled, a list of all relevant airports is displayed, along with the winds, QNH and calculated Transition Level.

smart_pdc

true

If enabled, the plugin overwrites the default clearance aliases to automatically include the correct airborne frequency.

custom_colours

false

If enabled, the plugin will use the colours defined in the settings file for tag items and screen elements.

Colour Settings

Below is a table of all available colour settings, including a brief description of the setting and the values used by the plugin when logged on an Shannon or a Dublin ACC position.

The custom_colours setting must be set to true for any of the settings below to take effect.

Only one colour can be defined per colour setting. The plugin will not dynamically change colours depending on your position if custom colours are used.

Colour Name

EISN

EIDW

Description

colour_ok

Colour used for tag items when the information displayed does not requires controller action

colour_alert

Colour used for tag items when a process is ongoing or to alert that controller action may become necessary

colour_bad

Colour used for tag items when the information displayed requires controller action

colour_background

Colour used for the background of plugin screen elements

colour_highlight

Colour used for the background of highlighted plugin screen elements

colour_border

Colour used for the border of plugin screen elements

colour_text

Colour used for most of the text on tag items and all text on plugin screen elements

colour_assumed

Colour used for the intention code tag item, when an aircraft is assumed

colour_coordinated

Colour used for the intention code tag item, when an aircraft is coordinated

colour_notified

Colour used for the intention code tag item, when an aircraft is notified (concerned)

colour_non_concerned

Colour used for the intention code tag item, when an aircraft is non-concerned

sweatbox_mode false
metar_notifications true
metar_display true
smart_pdc true
custom_colours false

colour_alert 240,225,41
colour_bad 255,152,0
colour_background 36,41,45
colour_highlight 78,85,89
colour_border 0,0,0
colour_text 200,200,200
colour_assumed 220,220,220
colour_coordinated 150,215,150
colour_notified 110,153,110
colour_non_concerned 135,128,118
